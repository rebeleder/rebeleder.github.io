忽然拾起来的git学习

昨天晚上忽然想给自己的博客加了子页面把最近看到的五笔练习网站和之前看到的图片合成网站放上去。不会git是做不到的。于是我又重新学起了git。

  1. git分为工作区（当前目录），index/stage区，本地库和远程库。

  2. 在工作去添加了文件或做了修改后，在工作区先用git add把修改加到stage区，如果不想一个一个文件的输入名字可以用git add .来全部添加。

  3. 然后用git commit -参数 来添加到本地库里。最好每次都用参数-m " ",意思是messenger，即这次修改的内容是什么，好让自己或别人知道都修改了什么。

  4. 要先设置commit时用的用户名和邮箱，用
    git config --global user.name ""
    git config --global user.email ""
    最好设置成和github 的一样的。

  5. 本地库和github的通信需要用到ssh密钥。一个电脑只需要一个密钥就够了。不同电脑是不同的密钥。用shh keygen -t rsa -C " "来生成密钥，生成的密钥在windows用户文件夹下的.shh文件夹里。密钥是一对，一个公钥一个私钥，我们只需要公钥来连接github。用cat ~/.ssh/id-ras.pub在终端里打印出密钥。复制后放到github个人设置的ssh设置里。然后在终端输入 ssh -T git@github.com来检查是否连接成功，出现用户名就是成功了。

  6. 关联本地库和远程库有两种方式，一是没有本地库的情况下，直接git clone 。二是有本地库情况下，可以关联本地库和远程库，用git push remote git@github.com:用户名/仓库名.git

  7. 这个时候就可以将本地库推到远程库了，如果远程库是空的，直接用推送就可以了。用git push 远程仓库名 本地分支名:本地分支名来推送到远程库。（如果本地分支和远程分支名字一样可以用写成一个，远程仓库名字默认为origin，本地分支默认为master，可以用git branch 查看。）可以用参数 -u 来关联本地分支和远程分支，-u 意为upstream 。

  8. 如果远程库不是空的，就有点麻烦了，因为远程库和本地库历史不一定一样。我就出现了这种问题，第一次还好用git pull 拉了远程库历史，然后push成功，第二次怎么都不行了。索性直接用 git push -f origin master 强制推送了，缺点就是历史commit没了。
